# Security Audits

This directory contains periodic security audit reports generated by AI models to assess the overall security posture of the ts-restic-server application.

## Purpose

AI-assisted security audits provide an automated, repeatable way to identify potential vulnerabilities, misconfigurations, and security best-practice violations across the codebase. These audits complement — but do not replace — manual security reviews and penetration testing.

## Process

- Audits are conducted regularly by feeding the full application source code to large language models acting as security auditors.
- Each audit analyzes API handlers, middleware, storage backends, configuration handling, dependencies, and deployment artifacts.
- Findings are categorized by severity (Critical, High, Medium, Low, Informational) and include file references, impact assessments, and remediation suggestions.

## Structure

Reports are organized by date:

```text
audits/
  2026-02-25/
    report.md            # Full audit report with all findings
    risk-assessment.md   # Maintainer risk assessment and decisions
  ...
```

## Disclaimer

AI-generated audits may contain false positives or miss issues that require runtime analysis, dynamic testing, or deep domain expertise. Findings should always be verified by a developer before acting on them. These reports serve as an additional layer of defense-in-depth, not as a certification of security.
